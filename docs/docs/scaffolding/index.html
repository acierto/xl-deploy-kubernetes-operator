Scaffolding

Steps to generate Deploy scaffolding using Helm Charts:

note

Before you proceed you should be connected to your Kubernetes cluster

  1. Create a folder namely xld
  2. Switch to xld folder 
cd xld
  1. Clone Deploy Helm chart repository 
git clone git@github.com:xebialabs/xl-deploy-kubernetes-helm-chart.git
  1. Switch to the cloned repository folder
cd xl-deploy-kubernetes-helm-chart
  1. Remove the values file for HAProxy as we will generate the scaffolding with NGINX support
rm -f values-haproxy.yaml
  1. Rename the values file for NGINX to values.yaml
mv values-nginx.yaml values.yaml
  1. Update the dependency for the charts
helm dependency update .
  1. Remove the lock file generated during updates of charts
rm -f Chart.lock
  1. Switch back to the parent folder, you should be in 'xld' folder
cd ..
  1. Create the Helm package using the available charts
helm package xl-deploy-kubernetes-helm-chart
  1. Remove the folder containing helm charts
rm -rf xl-deploy-kubernetes-helm-chart
  1. Rename the tgz file to xld.tgz
mv digitalai-deploy-10.1.tgz xld.tgz
  1. Use the operator sdk to initiate Deploy operator project, please make sure you are connected to your Kubernetes cluster otherwise you will get an error
operator-sdk init --domain digital.ai --plugins=helm
  1. Create APIs with 'xld' group and using the xld helm chart
operator-sdk create api --group=xld --version=v1alpha1 --helm-chart=xld.tgz
  1. We now need to generate the operator controller image. Export one variable to create the image along with a tag:
export OPERATOR_IMG="docker.io/xldevdocker/deploy-operator:1.0.0"
  1. Using the Makefile generated by operator sdk, build and push the image to the Docker Hub registry
make docker-build docker-push IMG=$OPERATOR_IMG
